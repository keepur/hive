### Task 4: Track 10 — Keychain fallback in config.ts + wizard honeypot calls

**Files:**
- Modify: `src/config.ts` (add Keychain fallback helper + modify required/optional)
- Modify: `src/setup/wizard.ts` (add honeypot calls in doSlack and doAnthropic)

Two changes: (1) config.ts should try Keychain when .env is missing a key, (2) wizard should store credentials in Keychain via honeypot alongside .env.

**Ordering in config.ts:** The file's execution order is: imports → dotenv.config (line 11) → `required()`/`optional()` function definitions (lines 13-21) → hive.yaml parse (lines 38-42) → `instanceId` const (line 47) → `config` export object (line 70+, which is where `required()`/`optional()` are first **called**). Since `fromKeychain` is only **called** at line 70+, it can safely reference `instanceId` as long as it's **defined** after line 47. Use a `function` declaration (hoisted) so `required()`/`optional()` can reference it by name even though it appears later in the file.

- [ ] **Step 1:** Add the `execFileSync` import to `src/config.ts`

Add to the existing imports at the top of the file:

```typescript
import { execFileSync } from "node:child_process";
```

- [ ] **Step 2:** Add `fromKeychain` as a `function` declaration after `instanceId` (after line 48)

Place this immediately after the `const instanceId = ...` line:

```typescript
/** Try to read a credential from macOS Keychain (honeypot-stored). */
function fromKeychain(key: string): string {
  if (process.platform !== "darwin") return "";
  try {
    return execFileSync(
      "security",
      ["find-generic-password", "-s", `hive/${instanceId}/${key}`, "-w"],
      { encoding: "utf-8", stdio: ["pipe", "pipe", "pipe"] },
    ).trim();
  } catch {
    return "";
  }
}
```

This is safe because: `instanceId` is a `const` defined on the line above, and `fromKeychain` is only **called** from within `required()`/`optional()` which are only **called** starting at line 70+ (inside the `config` export). The `function` declaration is hoisted so `required()`/`optional()` can reference the name.

- [ ] **Step 3:** Modify `required()` and `optional()` to try Keychain

Change `required()`:

```typescript
function required(key: string): string {
  const val = process.env[key] || fromKeychain(key);
  if (!val) throw new Error(`Missing required env var: ${key}`);
  return val;
}
```

Change `optional()`:

```typescript
function optional(key: string, fallback: string): string {
  return process.env[key] || fromKeychain(key) || fallback;
}
```

- [ ] **Step 4:** In `src/setup/wizard.ts`, add honeypot calls after saving to .env

In `doAnthropic()`, after the `saveEnv(env)` call, add:

```typescript
  // Also store in Keychain for honeypot-based credential flow
  try {
    execFileSync("honeypot", ["set", "ANTHROPIC_API_KEY", env.ANTHROPIC_API_KEY], {
      stdio: ["pipe", "pipe", "pipe"],
    });
  } catch {
    // honeypot may not be on PATH (dev mode) — .env is sufficient
  }
```

In `doSlack()`, after the `saveEnv(env)` call near the end of the function, add:

```typescript
  // Also store in Keychain for honeypot-based credential flow
  try {
    if (env.SLACK_APP_TOKEN) {
      execFileSync("honeypot", ["set", "SLACK_APP_TOKEN", env.SLACK_APP_TOKEN], {
        stdio: ["pipe", "pipe", "pipe"],
      });
    }
    if (env.SLACK_BOT_TOKEN) {
      execFileSync("honeypot", ["set", "SLACK_BOT_TOKEN", env.SLACK_BOT_TOKEN], {
        stdio: ["pipe", "pipe", "pipe"],
      });
    }
  } catch {
    // honeypot may not be on PATH — .env is the primary store
  }
```

Note: `execFileSync` is already imported in `wizard.ts`. `honeypot` is only on PATH after global npm install — the try/catch handles dev mode gracefully. All calls use `stdio: ["pipe", "pipe", "pipe"]` to avoid leaking credentials to terminal output.
