# Pull-Based Deploy Checker

## Overview

Replace the push-based GitHub Actions deploy pipeline with a pull-based approach. A launchd job on the Mac Mini runs every 5 minutes, checks if the `deploy` branch has new commits, and runs the existing `deploy.sh` if so.

## Deploy Checker Script

New file: `service/deploy-check.sh`

```
1. cd ~/build/hive
2. git fetch origin deploy --quiet
3. Compare local HEAD vs origin/deploy
   → If equal: exit 0 (nothing to do)
4. Run ~/services/hive/service/deploy.sh
```

The script delegates all deploy logic to the existing `deploy.sh` — pull, check, build, backup, rsync, restart, health check, rollback, notify. The checker only decides *whether* to deploy.

### Logging

Stdout/stderr go to `~/services/hive/logs/deploy-check.log` (via the launchd plist). The log captures both "nothing to do" skips and full deploy output. Rotated by the existing `rotate-logs.sh`.

### Concurrency

The launchd plist uses `StartInterval` (every 300 seconds). If a previous run is still going (deploy in progress), launchd skips the interval — it won't stack runs.

## Launchd Plist

Label: `com.hive.deploy-check`

Generated by `setup/generate-plist.ts` alongside the existing two plists. Uses `StartInterval` of 300 (5 minutes). Points to `${DEPLOY_DIR}/service/deploy-check.sh`.

Installed by `service/install.sh` — symlinked, bootstrapped, and managed alongside the existing plists.
